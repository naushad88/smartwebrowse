Plugin Overview

The Network Merchants Inc Gateway (NMI) for WooCommerce is a comprehensive payment processing solution that enables your WooCommerce store to accept credit card payments securely through Network Merchants Inc. (NMI).

Key Features

  • PCI DSS Compliant - Secure tokenization without storing sensitive card data
  • WooCommerce Blocks Support - Modern checkout experience with React-based payment forms
  • HPOS Compatible - Works with WooCommerce High-Performance Order Storage
  • Refund Support - Process full or partial refunds directly from WooCommerce
  • Debug Logging - Comprehensive transaction logs for debugging and monitoring
  • Collect.js Tokenization - PCI DSS compliant payment processing

Prerequisites

⚠️ Important: Ensure your WordPress site meets these requirements before installation.

System Requirements

  • WordPress: Version 5.8 or higher
  • WooCommerce: Version 5.0 or higher
  • PHP: Version 7.4 or higher (PHP 8.0+ recommended)
  • MySQL: Version 5.6 or higher
  • SSL Certificate: Required for secure payment processing

Required Accounts

  • NMI Merchant Account: Active Network Merchants Inc merchant account
  • API Credentials: Username, Password, and Security Key from NMI
  • Collect.js Account: For PCI DSS compliant tokenization

Installation Steps

Step 1: Download the Plugin

After purchasing the plugin, you'll receive a ZIP file named:

network-merchants-inc-gateway-for-woocommerce.zip

Step 2: Upload to WordPress

Method A: WordPress Admin Dashboard

  1. Log in to your WordPress admin dashboard
  2. Navigate to Plugins → Add New
  3. Click "Upload Plugin"
  4. Choose the ZIP file and click "Install Now"
  5. Click "Activate Plugin" after installation

Method B: FTP Upload

  1. Extract the ZIP file on your computer
  2. Upload the extracted folder to /wp-content/plugins/
  3. Activate the plugin from WordPress admin dashboard

Step 3: Verify Installation

✅ Success! The plugin should now appear in:

WooCommerce → Settings → Payments → Network Merchants Inc Gateway

Configuration Guide

Step 1: Access Payment Settings

  1. Go to WooCommerce → Settings → Payments
  2. Find "Network Merchants Inc Gateway"
  3. Click "Manage" or "Configure"

Step 2: Configure Basic Settings

Enable/Disable:

  • ✅ Check "Enable Network Merchants Inc Gateway"
  • ✅ Check "Enable for virtual products" (if applicable)

Step 3: API Configuration

🔒 Security Note: Keep your API credentials secure and never share them publicly.

Required API Settings:

  • Security Key: Your NMI security key
  • Collect.js Public Key: Your Collect.js public key

Step 4: Test Mode Configuration

For Testing:

  • ✅ Enable "Test Mode" during initial setup
  • Use test card numbers: 4242 4242 4242 4242
  • Use any future expiration date and any 3-digit CVC

Step 5: Advanced Settings

Optional Configurations:

  • Debug Logging: Enable for troubleshooting
  • Order Status: Set completion status (default: Processing)
  • Payment Method Title: Customize checkout display name
  • Payment Method Description: Add helpful text for customers

Testing Instructions

Step 1: Enable Test Mode

Before going live, always test the payment gateway:

  1. Ensure "Test Mode" is enabled in plugin settings
  2. Save the configuration

Step 2: Test Payment Processing

Test Card Numbers:

  • Visa: 4242 4242 4242 4242
  • Mastercard: 5555 5555 5555 4444
  • American Express: 3782 822463 10005
  • Declined Card: 4000 0000 0000 0002

Use any future expiration date and any 3-digit CVC

Step 3: Verify Transaction Logs

After test transactions, check:

  1. Go to WooCommerce → Orders
  2. Verify test orders are created
  3. Check order status and payment details
  4. Review debug logs if enabled

Step 4: Disable Test Mode

✅ Ready for Production!

Once testing is complete:

  1. Disable "Test Mode"
  2. Save configuration
  3. Your store is ready to accept real payments

Troubleshooting

Common Issues & Solutions

Payment Declined

Possible Causes:

  • Incorrect API credentials
  • Test mode enabled with live credentials
  • Invalid card information
  • Insufficient funds

Solution: Verify API settings and test with different card numbers

Plugin Not Appearing

Possible Causes:

  • Plugin not activated
  • WooCommerce not installed/activated
  • PHP version incompatibility

Solution: Check plugin activation and system requirements

SSL Certificate Issues

Issue: Payment forms not loading or SSL errors

Solution: Ensure your site has a valid SSL certificate

Debug Logging

Enable debug logging for detailed error information:

  1. Go to plugin settings
  2. Enable "Debug Logging"
  3. Check logs at: wp-content/uploads/wc-logs/
